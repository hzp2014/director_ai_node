## 应用概览

### 应用信息

- **应用名称**: AI漫导 (DirectorAI)
- **版本**: 1.0.0
- **包名**: com.directorai.director_ai
- **平台**: Android
- **最低系统要求**: Android 5.0+ (API 21+)

### 核心功能

AI漫导是一款基于AI智能体架构的漫剧制作应用，能够：
- 📝 智能生成剧本和场景
- 🖼️ 自动生成分镜图片
- 🎬 合成场景视频
- 🎞️ 合并完整视频
- 💾 保存到手机相册

## 剧本创作流程

### 完整流程图

```
用户输入创意
    ↓
GLM-4.7 理解需求
    ↓
生成剧本大纲
    ↓
确认剧本
    ↓
生成分镜图片
    ↓
生成分镜视频
    ↓
合并完整视频
    ↓
保存到相册
```

### 步骤详解

#### 步骤 1: 输入创意

**操作方法**:
1. 打开应用
2. 在底部输入框中输入创意描述
3. 点击发送按钮

**创意示例**:
```
"制作一个雪地里的冒险故事，主角是一个勇敢的小男孩"
```

```
"樱花树下，女孩在读书的唯美场景，10秒钟"
```

```
"一个温馨的家庭聚餐场景，三代同堂"
```

#### 步骤 2: AI 处理

AI 会自动执行以下操作：
- 理解用户需求
- 分析场景要素
- 规划角色设定
- 生成剧本大纲

**状态显示**:
- 🔄 "正在理解你的创意..."
- 📝 "正在生成剧本..."
- ✅ "剧本生成完成！"

#### 步骤 3: 剧本确认

系统会进入剧本确认页面，用户可以：
- 查看剧本详情
- 修改场景内容
- 确认或重新生成

**重新生成按钮** (右上角):

- 可以附带修改建议

### 操作流程

#### 确认剧本

1. 检查剧本内容是否满意
2. 查看角色设定是否准确
3. 查看每个场景的旁白和情绪
4. 如需修改，在底部输入框输入建议
5. 点击"确认并继续生成"按钮

#### 修改单个场景

1. 找到要修改的场景
2. 点击"编辑提示词"按钮
3. 在弹出的对话框中修改提示词
4. 点击"保存"
5. (可选) 点击"重新生成"按钮重新生成该场景

#### 重新生成剧本

1. 点击右上角"重新生成"按钮
2. (可选) 在底部输入框输入修改建议
3. 确认后系统会重新生成整个剧本

## 分镜生成

### 生成过程

确认剧本后，系统会自动开始生成分镜图片：

```
🎬 正在生成分镜图片...
    ↓
场景 1: 正在生成... ████░░░░ 40%
    ↓
场景 2: 正在生成... ████████░ 80%
    ↓
场景 3: 已完成 ✓
    ↓
...
```

### 进度显示

在剧本卡片中会显示：
- 🔴 待生成 (灰色)
- 🟡 生成中 (黄色进度条)
- 🟢 已完成 (绿色勾选)
- ❌ 生成失败 (红色叉号)

### 生成内容

每个场景会生成：
- 🖼️ 分镜图片 (1024x1024)
- 📝 图片生成提示词
- 📊 生成参数 (风格、角度等)

### 查看分镜图片

生成完成后：
1. 点击剧本卡片
2. 进入剧本详情页
3. 查看每个场景的分镜图片
4. 可以放大、缩小查看

### 单独生成场景

如果某些场景生成失败或不满意：

1. 找到对应场景
2. 点击[重试]按钮
3. 系统会重新生成该场景

## 视频生成

### 自动生成

所有分镜图片生成完成后，系统会自动开始生成视频：

```
🎬 正在生成分镜视频...
    ↓
场景 1: 正在生成... ████░░░░ 40%
    ↓
场景 2: 正在生成... ████████░ 80%
    ↓
场景 3: 已完成 ✓
    ↓
...
```

### 手动触发

如果想手动开始生成：

1. 确保所有场景的图片已生成
2. 在剧本卡片上找到"开始全部生成"按钮
3. 点击按钮开始生成所有场景的视频

### 生成参数

每个视频包含：
- 📹 视频时长 (默认5-10秒)
- 🎨 视频风格
- 📷 镜头运动 (平移、缩放、跟踪等)
- 🎵 背景音乐 (可选)

### 查看视频

生成完成后：
1. 点击剧本卡片展开
2. 查看每个场景的视频
3. 可以播放、暂停

### 视频操作

每个视频支持以下操作：

| 操作 | 图标 | 功能 |
|------|------|------|
| 播放/暂停 | ▶️ / ⏸️ | 播放或暂停视频 |
| 下载 | ⬇️ | 下载视频到本地 |
| 保存相册 | 📷 | 保存到手机相册 |
| 复制链接 | 🔗 | 复制视频链接 |
| 查看提示词 | 📋 | 查看生成提示词 |
| 重新生成 | 🔄 | 重新生成该场景视频 |

## 视频合并

### 合并条件

满足以下条件可以合并视频：
- ✅ 所有场景视频已生成
- ✅ 没有正在进行的生成任务
- ✅ 有足够的存储空间

### 合并操作

#### 方法 1: 在剧本卡片中

1. 等待所有场景视频生成完成
2. 找到"合并视频"按钮 (粉色渐变按钮)
3. 点击按钮
4. 在弹出的对话框中确认
5. 等待合并完成

#### 方法 2: 在设置页面

1. 进入设置页面
2. 找到"视频合并"卡片
3. 点击"合并场景视频"按钮
4. 选择要合并的剧本
5. 确认并开始合并

### 合并流程

```
1. 下载所有场景视频 (0% - 60%)
    ├── 场景 1: 下载中... 100%
    ├── 场景 2: 下载中... 100%
    └── ...
   ↓
2. 合并视频文件 (60% - 95%)
    ├── 使用 MediaMuxer 合并
    ├── 处理时间戳偏移
    └── 合并音频/视频轨道
   ↓
3. 清理临时文件 (95% - 100%)
    ↓
✅ 合并完成！
```

### 合并进度

合并过程中会显示进度对话框：

```
┌─────────────────────────┐
│  正在合并视频...        │
│  ████████░░ 60%        │  ← 进度条
│  下载场景 3/7...       │  ← 当前状态
└─────────────────────────┘
```

### 合并结果

合并完成后：
- 合并后的视频保存在应用内部
- 可以在设置页面查看
- 可以播放预览
- 可以保存到相册

### 合并统计

在设置页面可以看到：
- 已合并视频数量
- 占用存储空间

## 相册保存

### 保存图片

#### 保存分镜图片

1. 打开剧本详情
2. 找到要保存的场景图片
3. 点击图片
4. 点击"保存到相册"按钮
5. 图片保存到 `Pictures/Movies` 文件夹

#### 保存角色图片

1. 在剧本确认页面查看角色
2. 点击角色三视图
3. 点击"保存到相册"按钮

### 保存视频

#### 保存单个场景视频

1. 打开剧本详情
2. 找到要保存的视频
3. 点击视频下方的"保存"按钮
4. 视频保存到 `Movies/Movies` 文件夹

#### 保存合并后的完整视频

1. 在设置页面找到"视频合并"卡片
2. 点击已合并的视频
3. 点击"保存到相册"按钮

#### 保存网络视频

如果视频是网络链接：
1. 点击"保存到相册"按钮
2. 系统会先下载视频到临时目录
3. 下载完成后自动保存到相册

### 保存进度

保存过程中会显示进度：

```
📥 下载中... ████████░░ 80%
```

保存成功后会显示提示：

```
✅ 已保存到相册
```

### 保存位置

- 图片保存到: `Pictures/Movies/`
- 视频保存到: `Movies/Movies/`

### 权限说明

应用需要以下权限：
- ✅ 存储权限 (READ_EXTERNAL_STORAGE)
- ✅ 写入权限 (WRITE_EXTERNAL_STORAGE)

首次保存时会请求权限，请允许。

### API 配置

#### 支持的 API 服务

| 服务 | 用途 | 说明 |
|------|------|------|
| 🧠 智谱 GLM-4.7 | 智能决策 | 剧本生成、场景规划 |
| 🎥 视频生成 | 视频生成 | 场景视频生成 |
| 🖼️ 图像生成 | 图像生成 | 分镜图片生成 |
| 👁️ 豆包 ARK | 图片识别 | 图片内容识别 |

#### 配置步骤

1. 进入设置页面
2. 找到"API 配置"卡片
3. 点击要配置的服务
4. 在弹出的对话框中输入 API Key
5. 点击"保存"按钮

#### API Key 获取

**智谱 GLM-4.7**:
- 访问: https://open.bigmodel.cn/
- 注册账号
- 在控制台创建 API Key

**苍何 API** (推荐):
- 访问: https://api.canghe.ai/
- 注册账号
- 在控制台创建 API Key

**豆包 ARK**:
- 访问: https://ark.cn-beijing.volces.com/
- 注册账号
- 在控制台创建 API Key

#### 安全提示

⚠️ **重要**:
- API Key 保存在本地
- 不要泄露 API Key
- 定期更换 API Key
- 建议使用测试 Token 进行开发

## 常见问题

### 剧本生成

**Q: 如何修改已生成的剧本？**

A: 在剧本确认页面：
1. 点击右上角"重新生成"按钮
2. 在底部输入框输入修改建议
3. 确认后重新生成

**Q: 可以删除某些场景吗？**

A: 目前不支持删除场景，建议重新生成剧本。

**Q: 剧本生成失败怎么办？**

A: 
1. 检查网络连接
2. 检查 API Key 是否正确
3. 确认账户余额充足
4. 重试生成

### 分镜生成

**Q: 分镜图片生成很慢怎么办？**

A: 
1. 检查网络速度
2. 减少场景数量
3. 检查 API 服务状态
4. 使用其他 API 服务

**Q: 如何重新生成某个场景的图片？**

A: 
1. 打开剧本详情
2. 找到要重新生成的场景
3. 点击"重新生成"按钮

**Q: 图片生成失败？**

A: 
1. 检查 API Key 是否配置
2. 确认 API 服务可用
3. 查看错误信息
4. 重试生成

### 视频生成

**Q: 视频生成需要多长时间？**

A: 通常每个视频需要 30-60 秒，取决于视频时长和服务器负载。

**Q: 可以调整视频时长吗？**

A: 目前视频时长由系统自动决定，通常为 5-10 秒。

**Q: 视频生成失败？**

A: 
1. 确保图片已生成成功
2. 检查存储空间是否充足
3. 检查 API 服务状态
4. 重试生成

### 视频合并

**Q: 合并视频需要多少存储空间？**

A: 合并后的视频大约是各个场景视频大小的总和，建议预留 500MB 以上。

**Q: 合并失败怎么办？**

A: 
1. 检查存储空间
2. 确保所有场景视频已生成
3. 清理临时缓存
4. 重试合并

**Q: 合并后的视频质量如何？**

A: 使用无损合并技术，保持原始视频质量。

### 相册保存

**Q: 保存到相册失败？**

A: 
1. 检查存储权限是否已授予
2. 检查存储空间是否充足
3. 检查文件是否完整
4. 重试保存

**Q: 保存的图片/视频在哪里？**

A: 
- 图片: `Pictures/Movies/`
- 视频: `Movies/Movies/`

**Q: 可以批量保存吗？**

A: 目前不支持批量保存，需要逐个保存。

### 缓存管理

**Q: 缓存文件占用太多空间？**

A: 
1. 进入设置页面
2. 找到"缓存管理"
3. 点击"清理过期缓存"
4. 或点击"清空所有缓存"

**Q: 清理缓存会影响已保存的内容吗？**

A: 不会影响已保存到相册的内容，只删除临时缓存文件。

**Q: 缓存会自动清理吗？**

A: 会，系统会自动清理 2 天未访问的缓存文件。

### API 配置

**Q: 如何获取 API Key？**

A: 参考"设置功能"章节中的"API Key 获取"部分。

**Q: API Key 会被泄露吗？**

A: API Key 保存在本地应用中，不会上传到服务器，但建议不要分享设备。

**Q: 可以更换 API 服务吗？**

A: 可以，在设置页面随时切换不同的 API 服务。

### 其他问题

**Q: 应用崩溃怎么办？**

A: 
1. 重启应用
2. 清理缓存
3. 检查存储空间
4. 联系技术支持

**Q: 如何备份数据？**

A: 
1. 进入设置页面
2. 点击"导出数据 (JSON)"
3. 保存导出的 JSON 文件

**Q: 如何联系客服？**

A: 
1. 在应用内查看"关于"页面
2. 访问官方网站
3. 关注微信公众号"苍何"

| 权限 | 用途 | 是否必须 |
|------|------|---------|

| READ_EXTERNAL_STORAGE | 读取存储 | 是 |
| WRITE_EXTERNAL_STORAGE | 写入存储 | 是 |

### 性能优化建议

1. **生成速度**:
   - 使用稳定的网络连接
   - 选择高性能的 API 服务
   - 减少场景数量

2. **存储空间**:
   - 定期清理缓存
   - 及时保存重要内容到相册
   - 删除不需要的对话

3. **电池优化**:
   - 在充电状态下生成视频
   - 关闭不必要的后台应用
   - 使用省电模式会影响性能

### 隐私政策

1. **数据收集**:
   - 不收集用户个人信息
   - 不上传用户创作内容
   - 仅收集必要的匿名统计数据

2. **数据存储**:
   - 所有数据保存在本地
   - 缓存文件自动清理
   - 用户可随时删除数据

3. **第三方服务**:
   - API 调用通过第三方服务
   - 请参考第三方服务的隐私政策
